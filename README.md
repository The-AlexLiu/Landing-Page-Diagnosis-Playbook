# Landing Page Diagnosis Playbook

**落地页内容诊断：从投放承接到用户任务的经验沉淀**

过去看落地页时，很容易把问题说成“页面不够好看”“卖点不够突出”“按钮不够明显”。这些说法通常没错，但不够可执行，也很难说服投放、产品、设计、运营一起改。

我后来更倾向于把落地页当成一个“承接用户任务的页面”来看。

用户不是无缘无故来到页面的。他可能刚点了 Meta 素材，刚搜了一个 Google 关键词，刚从邮件里点进来，刚在站内看到一个 Banner，或者刚被销售发了一个链接。他点进来时已经带着预期：

- 这个东西是不是我刚才看到的那个？
- 它到底能解决我什么问题？
- 价格、风险、售后、隐私这些事情清不清楚？
- 我下一步应该买、注册、下载、咨询，还是先再看看？
- 这家公司、产品或页面值不值得信任？

所以我做落地页诊断时，不会先问“这个页面漂不漂亮”，而是先问：

> 这个页面有没有接住用户点进来时的那个问题？

这套方法不是只服务独立站商品页，也适合广告落地页、SaaS 官网、App 下载页、B2B 线索页、内容页、活动页和平台型产品页面。它的核心价值是把“页面不转化”拆成可检查、可排序、可验证的问题。

---

## 核心判断

落地页不是单独的一张销售页，而是转化链路中的一段。

用户看到的是一条连续链路：

> 入口承诺 → 页面首屏 → 内容解释 → 信任证据 → 行动路径 → 提交 / 购买 / 注册后的反馈

只要其中任何一段断了，转化就会掉。

| 表现 | 表面看起来 | 我会优先怀疑 |
|---|---|---|
| CTR 高但 CVR 低 | 素材很吸引人，页面不转化 | 广告承诺和首屏内容断裂 |
| 页面滚动深但按钮点击少 | 用户有兴趣但不行动 | CTA 不匹配用户阶段，或风险没有解除 |
| 加购高但购买低 | 商品有吸引力但订单少 | 运费、退换、价格、支付信任有问题 |
| 表单页访问多但提交少 | 线索质量差 | 表单过重、隐私说明弱、提交后收益不清楚 |
| Google Ads 落地页体验差 | 点击成本、转化率和投放质量受影响 | 关键词、广告、页面内容和导航没有对齐 |
| 用户问客服的问题很重复 | 页面内容不完整 | FAQ、价格、规则、售后信息没有前置 |

## 我会先看 4 个问题

### 1. 用户从哪里来

不同入口带来的用户预期完全不同。

| 来源 | 用户预期 | 页面最应该接住什么 |
|---|---|---|
| Google Search | 我主动搜索了一个问题或商品 | 关键词意图、具体答案、可比较信息 |
| Google Shopping / PMax | 我看到了商品图、价格、标题 | 同款商品、价格一致、库存和优惠 |
| Meta Ads | 我被素材角度吸引 | 素材里的痛点、场景、达人/UGC、优惠 |
| TikTok / Reels | 我被短视频情绪和画面打动 | 同款画面、快速利益点、移动端体验 |
| EDM / 私域 | 我已经对品牌有一点认知 | 活动规则、会员权益、专属优惠 |
| SEO 内容 | 我想了解或比较 | 结构化答案、可信来源、下一步推荐 |
| 销售发来的链接 | 我在评估方案 | 案例、价格、资质、交付边界 |
| 站内 Banner | 我已经在品牌站内 | 明确活动、商品集合或任务入口 |

### 2. 首屏能不能在几秒内讲清楚

首屏不需要说完所有信息，但必须让用户知道自己是否值得继续。

| 用户问题 | 页面应该回答 |
|---|---|
| 这是什么？ | 产品、服务、活动或任务入口 |
| 给谁用？ | 人群、角色、场景 |
| 解决什么问题？ | 用户利益，不只是品牌口号 |
| 为什么值得继续？ | 差异点、证据、结果、优惠 |
| 下一步做什么？ | 主 CTA，必要时给一个低承诺 CTA |

### 3. 用户为什么会不信

用户转化前会不断做风险评估：

- 这个评价是真的吗？
- 价格为什么是这个水平？
- 有没有隐藏费用？
- 不合适能退吗？
- 数据安全吗？
- 订阅能取消吗？
- 这个案例和我有什么关系？
- 我提交表单后会被频繁骚扰吗？

页面如果不主动回答这些问题，用户就会退出、比价、问客服、拖延，或者去搜品牌词。

### 4. 下一步是否顺手

CTA 不是按钮文案，而是用户承诺强度。

| CTA | 承诺强度 | 适用场景 |
|---|---:|---|
| 立即购买 / 下单 | 高 | 热流量、价格清楚、信任充分 |
| 预约 Demo / 联系销售 | 高 | B2B、高客单价、复杂决策 |
| 免费试用 / 注册 | 中 | SaaS、工具、平台产品 |
| 下载资料 / 查看案例 | 中低 | 冷流量、教育型页面 |
| 查看价格 / 对比方案 | 中 | 有兴趣但还在评估 |
| 观看演示 / 了解流程 | 低 | 新产品、复杂产品、冷启动 |

## 12 个诊断维度

为了让诊断不靠感觉，我把落地页拆成 12 个维度。每个维度都可以打 0 到 5 分，并记录截图、数据或页面证据。

| 维度 | 我主要看什么 | 常见问题 |
|---|---|---|
| 来源意图匹配 | 页面有没有接住广告、搜索词、入口承诺 | 广告讲优惠，页面讲品牌故事 |
| 首屏价值主张 | 用户几秒内能不能理解价值 | 标题抽象、首屏无 CTA、利益不清楚 |
| 信息架构 | 内容顺序是否符合用户决策 | 证据太后、规则太散、重点被埋 |
| 可扫描性 | 小标题、列表、表格是否好读 | 大段文字、卖点堆叠、没有层级 |
| 可信证据 | 有没有让用户相信的材料 | 评价空泛、案例无结果、数据无口径 |
| 产品/服务解释 | 用户是否理解具体怎么用 | 只写功能名，不写场景和结果 |
| CTA 与行动路径 | 下一步是否明确、顺手、可追踪 | 按钮多、文案虚、点击后无反馈 |
| 风险解除 | 用户的顾虑是否被提前回答 | 价格、售后、隐私、退款不清楚 |
| 移动端体验 | 手机上是否好看、好点、加载快 | 首屏白屏、弹窗遮挡、按钮太小 |
| 合规与透明 | 是否存在夸大、隐藏、诱导 | 隐藏费用、默认勾选、虚假稀缺 |
| 数据追踪 | 能不能判断问题出在哪里 | UTM 丢失、事件重复、归因口径不明 |
| 品牌一致性 | 页面是否和品牌、产品、客服口径一致 | 广告、页面、客服说法不一样 |

这个框架最重要的不是“总分多少”，而是找出哪几个问题正在阻塞用户继续往下走。

## 我的诊断流程

### 第一步：先还原用户进来前的预期

我不会一上来就改页面，而是先回到用户点击之前，看他是被什么信息带进来的。因为很多落地页问题不是页面里某个模块写得不好，而是入口承诺和页面承接没有对上。

| 我会看什么 | 为什么看 |
|---|---|
| 广告素材、关键词、邮件标题、站内入口 | 判断用户点进来时期待看到什么 |
| 用户所在渠道和设备 | 判断他是主动搜索、被素材吸引，还是已有一定认知 |
| 最终 URL 和跳转链路 | 确认 UTM、ref、gclid、fbclid 等参数没有丢 |
| 移动端首屏 | 还原大多数广告流量真正看到的第一屏 |

这一步的输出不是一堆建议，而是一句话判断：

> 这个页面有没有接住用户原本点进来的理由？

### 第二步：拆页面有没有完成“理解、相信、行动”

确认入口预期后，我会沿着用户阅读页面的顺序往下看。这里不只是看视觉，而是看页面有没有让用户完成三个动作：理解、相信、行动。

| 诊断层 | 我会看什么 |
|---|---|
| 理解 | 首屏是否讲清楚这是什么、给谁用、解决什么问题 |
| 相信 | 评价、案例、数据、价格、政策、资质是否足够可信 |
| 行动 | CTA 是否顺手，表单/购买/注册路径是否清楚 |
| 风险 | 运费、退款、隐私、订阅、隐藏费用、夸大承诺是否提前说明 |
| 数据 | GA4、广告平台、Pixel、热图、录屏能否支持判断 |

我会把问题尽量写成可验证的判断，而不是主观评价。比如不写“页面说服力不够”，而是写：

> 广告素材强调限时优惠，但首屏没有出现优惠信息，用户需要滚动到第三屏才看到活动规则。这会增加确认成本，也可能拉低移动端 CVR。

### 第三步：把问题排成执行顺序

诊断最后一定要落到行动优先级。页面通常不会只有一个问题，如果所有建议都同时做，就很难判断到底是哪一次修改起了作用。

我会按这几个标准排序：

| 判断标准 | 说明 |
|---|---|
| 影响范围 | 是否影响主要流量、主要设备、主要转化路径 |
| 修复确定性 | 是明显错误，还是需要实验验证 |
| 业务风险 | 是否涉及审核、价格、隐私、退款、订阅等风险 |
| 执行成本 | 文案、结构、开发、埋点、设计分别需要多少资源 |
| 验证方式 | 能否用 CVR、CPA、ROAS、表单率、加购率、录屏等验证 |

最后的输出应该是一组可以推进的动作，而不是一句“建议优化落地页”。

## 我会如何给优化建议分优先级

我不会把所有建议都列成“建议优化”。这样对执行没有帮助。

| 类型 | 适用情况 | 示例 |
|---|---|---|
| 立即执行 | 明显影响转化、审核、追踪或用户完成任务 | 页面打不开、CTA 不可点、价格不一致、UTM 丢失 |
| 观察等待 | 数据还不够，或波动可能来自学习期 / 样本不足 | 新广告组刚上线 1 天，样本过少 |
| 小预算测试 | 有明确假设，但需要验证 | 改首屏标题、换证据顺序、减少表单字段 |
| 暂停或降预算 | 页面问题会浪费预算或带来风险 | 支付异常、移动端白屏、活动价不一致 |
| 需要补充数据 | 当前无法判断 | 缺 GA4、缺广告素材、缺转化定义、缺利润数据 |

如果是投放落地场景，我会优先用这个排序：

> 优先级 = 花费占比 × 转化损失 × 修复确定性 × 风险等级
